# Deep Insight: スペックカードの「役割」再深掘り

- 日付: 2026-07-03
- 経緯: 初回分析（2026-07-03-spec-card-ux.md）は役割を「名刺 兼 戦績カード」と結論し、
  ベストラップ刻印のA〜D案まで進めた。ユーザーより「役割について今一度深掘りを」との要請。
- 本レポートは配置案（How）ではなく役割（What/Why）を再検討する。

## 1. 前回結論の自己批判

「名刺 兼 戦績カード」は**観察者を「所有者とその仲間」に限定したときだけ成立する結論**だった。
SpecCard は5つの文脈（車両一覧 / 比較ビュー / 共有カード・詳細 / 公開LP / 共有画像）に埋め込まれており、
「誰が・何を知りたくて見るか」で役割が変わる。単一の役割定義はそもそも問いの立て方が粗かった。

## 2. 観察者×文脈で分解すると、カードは4つの役割を持つ

| 文脈 | 観る人 | 頭の中の問い | カードの役割 |
|---|---|---|---|
| ガレージ（車両一覧） | 所有者本人 | 「俺の車、いいな」 | **鏡** — 愛着と投資を映す所有物 |
| SNS・走行会パドック | 仲間・同好者 | 「どんな車？その車高調どこの？」 | **名刺** — 自己紹介と会話の種 |
| 共有データ閲覧・（将来）購入 | 他のドライバー | 「このデータは自分の車に使えるか」 | **商品ラベル** — データの前提条件・比較可能性の担保 |
| 公開LP・共有画像 | 未登録者 | 「なにこれ、ちょっといいな」 | **広告塔** — 獲得チャネルの顔 |

重要なのは、**裏面の改造明細（パーツ名×メーカー）はパドックの定番会話「そのパーツどこの？」への回答そのもの**であり、
名刺役割と商品ラベル役割の両方に効く点。また商品ラベル役割は事業計画の堀
（検証済みデータ資産・「サーキット×車種」の比較可能性）に直結し、**Phase 0c（Give-to-Get）以降は主役になる**。
前回分析はこの商品ラベル役割を過小評価していた。

セットアップシート（空気圧・アライメント＝走行ごとに変わる可変データ）とカード（車両の不変〜緩変データ）の
分離は現設計の正しい骨格であり、カードは「可変データを解釈するための文脈レイヤー」と言い換えられる。

## 3. すでに在る構造美: 表＝感情の面、裏＝信頼の面

現デザインは直感的に正しい二面性を持っている。
**表面はトレカ（誇り・共有＝感情・社会的ジョブ）、裏面は申告書（前提条件・比較＝認知ジョブ）**。
フリップという操作が「愛でる対象としての車」と「データ文脈としての車」の境界になっている。
これは偶然の産物かもしれないが、守るべき資産。以後の機能追加はこの二面性に沿って
「表に足すもの＝誇れるもの」「裏に足すもの＝信頼を作るもの」と振り分けるべき。

日本の車文化への写像: 表＝愛車自慢（オーナーズクラブ・パドック文化）、
裏＝中古車オークションの出品票（申告と評価点で信頼を作る文化）。

## 4. 発見した構造的ズレ（前回分析より深い問題）

### 4-1. レア度の軸が事業価値と直交している【最重要】
カードの色（NORMAL=灰 → FULL=金）は改造度、すなわち**お金の投入量**で決まる。
コードの意図は「装飾ではなく情報としての色」だが、金色は文化的に「上位・レア」と読まれる（意図と読解のギャップ）。
一方、事業が報いたい行動は**記録・検証・成長**（NSM=月1回以上記録するユーザー）。

この軸ズレの被害者が2セグメントいる:
- **ビギナー（流入源）**: ノーマル車は灰色の"一番つまらないカード"になる。最も歓迎すべき層に「君のカードは地味」と伝えてしまう
- **ワンメイク参戦者（最高価値データの生産者）**: 全員ほぼ同仕様・ほぼノーマルなので全員同じ灰色カードになる。
  戦績文化が最も強い層に、誇る場所を一切与えていない

改造度カラー自体は「データの前提条件」として正しい情報なので廃止すべきではない。
問題は**それが唯一のレア度軸である**こと。お金で買えない第二の軸（検証済み実績）が必要。

### 4-2. カードが静的で、記録行動と接続していない
現カードが変化するのはパーツを買った時だけ。走っても記録しても何も変わらない。
「成長していく」というプロダクトテーマと NSM のどちらにも接続していない。
戦績を載せる決定は単なる情報追加ではなく、**静的な証明書を「育つ生き物」に変える決定**であり、
版管理（共有スナップショットの鮮度・更新タイミング）とセットで考える必要がある（旧仮説A/Bが不可分な理由）。

### 4-3. トレカの形は「集める」を約束するが、集める仕組みがない
63:88 という実物トレカ比率は収集・交換文化の記号であり、ユーザーに無意識の期待を作る。
現状その約束は未回収（自分の1〜3枚を眺めるだけ）。
回収先として最も筋が良いのは**走行会でのカード交換**（「今日のパドック」アルバム）。
GTM最重要チャネル（走行会B2B2C）と Give-to-Get 文化醸成にそのまま重なる。今すぐ作る必要はないが、
カードの情報設計はこの未来（他人のカードを保存・閲覧する）を壊さない形にしておくべき。

### 4-4. 「申告」で止まっていて「検証」への道筋がカード上にない
現カードの信頼要素は申告値表記のみ。事業の商品規格は「ラップタイム証憑付きデータ」。
類推: トレカ文化の PSA グレーディング、中古車文化の出品票評価点 — どちらも
**第三者検証がレア度と価格を決める**。ロガー証憑はまさにこの「鑑定」に相当する既存資産であり、
カードに検証マークが載って初めて、売り手（P4）の実績証明と買い手の品質判断が機能する。

## 5. 役割の時間発展（フェーズ対応）

| フェーズ | 主役の役割 | カードに必要なもの |
|---|---|---|
| Phase 0（現在） | 鏡・名刺・広告塔 | 誇れる要素（戦績・成長）、共有動線、フリップ発見性 |
| Phase 0c（共有） | 商品ラベルが台頭 | 前提条件の完全性、匿名化との両立 |
| Phase 1（売買） | 商品パッケージ | 検証グレード表示、売り手信頼、（一覧で使える）小型表現 |
| Phase 2（集合知） | ベンチマーク文脈 | 同一マス内での自車位置の示唆 |

## 6. 新たに導かれる仮説（旧A〜Dへの含意つき）

- **仮説E: サーキットデカール** — 記録した走行のあるサーキットのステッカー風マークがカードに貼られていく。
  実車が主催者ステッカーやパドックパスを重ねていく文化の写し。**お金で買えず、記録行動でしか増えない**ため
  4-1の軸ズレを直接是正し、NSMをそのままゲーム化する。ノーマル灰色カードにも誇りが宿る。
  低情報密度の原則と衝突しないよう、個数上限や裏面配置を検討。サーキットDB（既存の差別化資産）を活用。
- **仮説F: 検証グレードの「箔」** — ロガー証憑付きベストを持つカードにのみ、鑑定シール（または箔・ホロ調の控えめな加工）。
  改造度カラーと独立した第二のレア度軸。4-4への回答であり、売買フェーズの品質表示の先行実装になる。
- **旧A〜Dの再解釈**: 表＝代表ベスト1本（＋証憑シール）、裏＝サーキット別戦績表という**A＋D構成は
  「表＝感情/裏＝信頼」の二面性から理論的に導ける**。B（フッター計器化）とC（ヒーロープレート）は
  表面の感情的主役を車からタイムに移す案であり、役割モデル上はやや過剰と再評価。
- **配慮が必要な点**:
  - 遅いタイムは誇れない — ビギナーには絶対タイムより**成長差分（例: 今季 −2.3s）**の方が感情ジョブに合う
  - ワンメイクの機密性 — レース前に手の内を晒せない。共有面での戦績表示は**車両ごとにopt-in**が前提
  - 他者比較の演出はコミュニティの萎縮を招き得る。表面はあくまで自己参照（自己ベスト・自己成長）をデフォルトに

## 7. ユーザーへの確認事項

1. **カードの主語**: 「車両」のカードか、「ドライバー×車両」のカードか。戦績は対に属する
   （同じ車でもドライバーが違えば別の戦績）。ownerLabel の扱い・売り手実績との接続に影響する
2. **レア度の第二軸**: 「検証済み実績（証憑・デカール）」をお金で買えないレア度として導入する方針に賛成か
3. **表面のヘッドライン**: 絶対ベスト / 成長差分 / 所有者が選択式 — どれをデフォルトにするか
4. **収集の約束**: 走行会カード交換の構想は Phase 0 のうちに種を蒔くか（情報設計だけ将来対応にしておくか）
