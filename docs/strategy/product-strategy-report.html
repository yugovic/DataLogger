Solution Brief

実車セットアップデータマーケットプレイス

Problem Recap

サーキット走行のセットアップは試行錯誤に時間と金がかかる（走行1回2-3万円、タイヤ数万円）。しかし「同じ車種×同じサーキットの検証済みセットアップ」にアクセスする手段がない。シムでは売買文化が成立済みだが、実車では世界中に C2C マーケットプレイスが存在しない。

Key Features

1. セットアップ記録（無料・無制限）

タイヤ空気圧4輪・サスペンション・アライメント等を詳細に記録。未入力はnull保存。記録の継続が在庫の生産ライン。

2. Give-to-Get 共有

自分のデータを共有した人だけが他人のデータを閲覧可能。相互性でコールドスタートを解決。

3. ロガー連携・ラップタイム証憑

AIM CSV / デジスパイス NMEA でラップタイム証拠を紐付け。データの信頼性を証憑で担保。

4. C2C マーケットプレイス

Stripe Connect で手数料15-20%のみ受領。資金移動業登録を回避。売り手が価格決定。

5. 比較ビュー

購入データと自分のデータの差分をハイライト表示。「何を変えれば速くなるか」を視覚化。

Step 5 of 5

ADR-001

Stripe Connect によるC2Cデータ売買決済基盤の採用

Proposed 2026-06-26 — Deciders: 創業者

Context

Business

  • 資金移動業の登録を回避したい
  • 単価 ¥500-3,000 の少額決済
  • ブートストラップ — 初期費用最小化

Technical

  • 単独開発（バス係数1）
  • React + Firebase スタック
  • Phase 2 サブスクにも同一基盤

Legal

  • C2C代金の一時預かり = 資金移動業リスク
  • Stripe Connect ダイレクトペイメントで回避可能

Decision

Stripe Connect（Standard accounts） をC2Cデータ売買の決済基盤として採用する。 買い手→売り手にダイレクトペイメント、プラットフォームは application_fee_amount で手数料（15-20%）のみ徴収。

Consequences

Positive

  • 資金移動業を構造的に回避
  • Stripe SDK の成熟度 — 単独開発でも実装可能
  • グローバル対応 — 海外展開時も同一基盤
  • Stripe Billing で Phase 2 サブスクにスムーズに移行
  • 売り手が自分のダッシュボードで売上確認可能

Negative

  • Standard account のKYCが売り手の離脱ポイント
  • Stripe一本化によるプラットフォーム依存
  • 基本手数料3.6%が少額決済の利益を圧迫

Alternatives Considered

AlternativeVerdictReason
自前決済（銀行振込+手動分配） Rejected 手動運用が単独体制で持続不可能。資金移動業リスクあり。H3手動テストのみに使用可
PAY.JP Marketplace Rejected Marketplace API のドキュメント・事例がStripeに比べ少ない。海外展開時の対応不明
Stripe Connect（Express） Rejected 売り手が自分のダッシュボードを持てず透明性が低い。売り手の信頼が生命線
