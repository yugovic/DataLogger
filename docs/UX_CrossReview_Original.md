# Velocity Logger UX クロスレビュー

## 1. 検証済み：前回の指摘が正しい点

- **`/CarSetup.tsx` の「新規コピー」ボタンに `?copy=` パラメータが不足**
  - `SetupCard.tsx` 側のURL生成に問題がある可能性もあるため、併せて確認が必要。
  - 優先度：**高**

- **`CarSetup` 本体に未保存データ保護がない**
  - `VehicleModal.tsx` では `beforeunload` とダーティ判定が実装されているが、メインのセットアップ入力画面には未実装。
  - 長時間の入力が失われるリスクが高い。
  - 優先度：**高**

- **「前回読み込み」と「前回から引き継ぎ」の使い分けが分かりにくい**
  - `handleLoadPrevious` は前回セッションの全フィールドをコピー。
  - `handleInheritFromPrevious` は車種で絞り込み、セッション非依存項目だけをコピー。
  - UIラベルだけではユーザーが誤って使い分ける可能性あり。
  - 優先度：**中〜高**

- **オンボーディングで収集した目標データが活用されていない**
  - `OnboardingWizard.tsx` の `handleComplete` で `homeCircuit`, `goalType`, `targetLapTime` を保存するが、ダッシュボードやセットアップ画面で参照・活用されていない。
  - モチベーション維持と目標導線のUX機会損失。
  - 優先度：**中**

## 2. 追加で発見したUX課題

### 2.1 編集時のコピーURLが機能しない可能性
- `SetupCard.tsx` の「コピー」アクションで `?copy=<id>` を付けたURLに遷移するが、`CarSetup.tsx` のURLパラメータ取得部分が確認されていない。
- もし `useSearchParams` や同等の取得が抜けていると、コピー機能が完全に壊れる。
- 優先度：**高**（確認・修正を推奨）

### 2.2 タブ間の未保存遷移に保護なし
- タブ切り替え（`BasicInfoTab` → `SuspensionTab` → `DrivingTab`）は同一コンポーネント内だが、ユーザーがブラウザバックやリロードすると状態が失われる。
- `beforeunload` だけでなく、`Prompt` や `Modal` で離脱確認を出すのが望ましい。

### 2.3 モバイル入力の操作性
- `SuspensionTab.tsx` では多数の `AutoComplete` と `StepNumber` が密集。
- スマホではキーボードとステッパーが干渉しやすく、タップミスや誤入力が起きやすい。
- 入力タイプごとにキーボードタイプ（`inputMode`, `type`）の最適化を検討。

### 2.4 シェア・公開設定の一貫性
- `ShareToggle.tsx` と `PublicShareButton.tsx` は似た機能を別UIで提供。
- 「公開シェア」と「内部共有」の概念が混在し、ユーザーがどちらを使えばよいか迷う。
- 用語統一と導線の整理が望ましい。

### 2.5 ダッシュボードの空状態
- `Dashboard.tsx` には「最初のセットアップを記録」CTAがあるが、目標ラップタイムやホームサーキットの表示がない。
- オンボーディング情報と連携することで、初回時の導線強化が可能。

## 3. 強み（引き続き保持すべき点）

- **タブ・入力コンポーネントの再利用性**：`BasicInfoTab`, `SuspensionTab`, `DrivingTab` の分割が明確で保守しやすい。
- **Zodバリデーション**：`setupService.ts` で保存前にスキーマ検証を行い、データ品質を守っている。
- **匿名化対応のシェア**：`setupService.ts` の `toggleSetupVisibility` でユーザー同意を伴った匿名化処理が実装されている。
- **テレメトリー連携**：ラップ比較やトレース保存がエンゲージメントを高める。
- **車両設定画面の未保存保護**：`VehicleModal.tsx` の `beforeunload` 実装は良い precedent になりうる。

## 4. 優先度まとめ

| 優先度 | 項目 |
|---|---|
| 高 | CarSetup の未保存保護を追加 |
| 高 | コピーURLの `?copy=` パラメータ取得・動作確認 |
| 高 | 新規コピーボタンのURLパラメータ修正 |
| 中〜高 | 「前回読み込み」vs「引き継ぎ」のUI説明強化 |
| 中 | モバイル入力体験の改善 |
| 中 | オンボーディング目標データのダッシュボード活用 |
| 中 | シェア・公開UIの概念統一 |
| 低 | タブ間遷移での状態保持確認 |

## 5. 推奨する次のアクション

1. `CarSetup.tsx` に `beforeunload` + ダーティ判定を実装。
2. `SetupCard.tsx` / `CarSetup.tsx` のコピーURL・パラメータ取得を統合して動作確認。
3. `handleLoadPrevious` / `handleInheritFromPrevious` のボタンにツールチップまたは説明文を追加。
4. `Dashboard.tsx` で `targetLapTime` / `homeCircuit` を表示し、目標導線を作る。
5. シェア系コンポーネントの名称・導線を1つに集約するデザイン検討。
