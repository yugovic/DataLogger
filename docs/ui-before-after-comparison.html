変更前の特徴

  • 中央配置の標準的なカードで、余白と影が汎用 UI に近い。
  • ブランド表現は文字ロゴ中心で、画面全体の情報階層は薄い。
  • アクセントは青ボタンに集中し、他の要素との関連が弱い。

変更後の特徴

  • スレート基調、太い数字/見出し、細い下部ストライプでアプリ全体の視覚言語を統一。
  • ロゴブロック、`Driver Access` ラベル、強い主ボタンで認証画面の階層を明確化。
  • 同じ設計方針を Header と Dashboard にも展開し、操作系画面としての密度を上げた。
