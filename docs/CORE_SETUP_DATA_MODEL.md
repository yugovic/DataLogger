# コア・セッティング記録モデル

## 目的

この画面の目的は車両スペックや改造履歴を網羅することではない。

1. その走行で実際に使った車両状態を残す
2. 後日、同じ状態を再現できる
3. 設定変更とラップ、タイヤ状態、ドライバーフィードバックを比較できる

項目を追加するかは、次の基準で判断する。

- 走行間で変更または選択される値か
- 走行開始時点で値を確定できるか
- 車両挙動、タイヤ、ブレーキ、ラップへ影響しうるか
- 単位、選択肢、測定方法をチーム内で揃えられるか

固定諸元、装着履歴、費用、メンテナンス履歴はコアのセッティング記録から外す。

## データの責務

### 車両管理

`Vehicle.setupConfig.adjustmentDefinitions` に、その車両で入力可能な項目を定義する。

- 安定した項目ID
- 分類
- 表示名
- 対象位置（車両全体、前後、各輪）
- 入力形式（数値、選択肢、文字、ON/OFF）
- 単位、最小値、最大値、刻み幅
- 選択肢、入力補足

ここに走行時の実値は保存しない。

### セットアップ記録

`CarSetup.adjustmentValues` に、そのセッションで実際に使った値だけを保存する。

定義IDに加え、表示名、単位、分類、位置、入力形式も保存時点でスナップショットする。車両側の定義が後から変わっても、過去記録を解釈できる。

### 専用フィールドを維持する項目

次の項目は分析・導出処理があるため、動的項目へ統合せず専用構造を維持する。

- 四輪の冷間・温間タイヤ空気圧
- 目標温間圧と次走行推奨値
- タイヤ銘柄、コンパウンド、前後サイズ
- 天候、気温、路面温度、湿度、気圧
- 走行距離、燃料量
- ラップタイム、ロガー証憑、テレメトリ参照
- ドライバーフィードバック、意図、結果、学び

## 動的項目の分類

- タイヤ: セットID、ヒートサイクル、残溝など
- ダンパー: 車両固有の調整軸、クリック、セットアップシート表記
- スプリング: 前後または各輪のレート
- 車高: 前後または各輪
- スタビライザー: 穴位置、段数、ブレード角
- アライメント: キャンバー、トー、キャスター
- ブレーキ: パッド、バイアス、ABS設定
- エアロ: ウイング角、フラップ位置
- 駆動系: デフ・プリロード、ランプ角など
- エンジン: ECUまたはBoP設定識別子
- 電子制御: ABS、TCなど
- 重量・バランス: バラスト、搭載位置、コーナーウェイト

## 実装済み

- 車両管理画面から動的項目を追加・削除可能
- 車両選択時に、その車両の項目だけをセットアップ画面へ表示
- 数値、選択肢、文字、ON/OFF入力に対応
- 未入力値を保存しない
- 保存、編集、複製、前回設定の引き継ぎに対応
- 定義変更後も保存時スナップショットから過去記録を表示
- 2セッション比較へ動的項目を追加
- 履歴カードへ主要3項目を表示
- 従来形式の車両・走行記録は従来UIで編集可能

## 実車デモ定義

- Toyota Yaris Cup Car: 6項目
  - タイヤセットID、ヒートサイクル
  - リア減衰力 1-14 click
  - フロントトー
  - 前ブレーキパッド、後ブレーキシュー
- Honda NSX GT3 EVO: 27項目
  - タイヤ、5-wayダンパー、スプリング、車高、ARB
  - アライメント、ブレーキ、ABS、TC、リアウイング
  - デフ、BoP設定、バラスト、各輪重量

NSXの5-wayダンパーは公開資料に各調整軸名と範囲がないため、現時点では前後それぞれをチームのセットアップシート表記で記録する。技術マニュアルを入手した場合は、同じ基盤上で調整軸ごとの数値項目へ分割する。

## 次に行うべき改善

1. 燃料量を走行開始時・終了時に分ける
2. タイヤの装着位置変更を追跡できるセット管理を追加する
3. 測定基準（車高測定点、トーの片側/合計など）を定義テンプレート化する
4. 同じ項目IDについて時系列グラフとラップ相関を表示する
5. 車種別テンプレートを用意し、定義を毎回手入力しなくても登録できるようにする
