# タイヤセット管理モデル

## 責務

- サービス側候補: 入力補助用の主要メーカー・製品名。完全な製品台帳にはしない。
- 所有タイヤセット: `tireSets` コレクションに物理セットを保存する。
- 車両: セット管理のON/OFF、対応サイズ、主に使うセットとの関連を持つ。
- セットアップ記録: 実際に使ったセットIDと製品情報のスナップショット、今回のヒートサイクル加算を保存する。

セットアップ記録は距離、周回数、使用回数の原本である。タイヤセットへ累積値を保存のたびに加算しない。

## 所有タイヤセット

主な項目:

- ユーザー表示用セットID
- メーカー、製品名、コンパウンド
- 前後サイズ
- 主に使用する車両
- 使用中、保管中、使用終了
- 使用開始日
- システム登録前の距離、周回数、ヒートサイクル

累積値は次の式で再計算する。

```text
累積値 = 登録前の初期値 + tireSetId が一致する全セットアップ記録
```

この方式により、走行記録の編集・削除・セット変更で二重加算しない。

## 旧データ互換

- `tireInfo.brand` は残し、新規保存ではメーカー名を入れる。
- 新規項目の `manufacturer`、`productName`、`tireSetId`、`tireSetCode` は任意。
- タイヤセット管理を使わない車両は、従来どおり自由入力できる。
- マスターや所有セットを後から変更しても、走行記録内の製品名・サイズは保存時スナップショットとして残る。
