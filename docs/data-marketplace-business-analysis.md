## 1. 構想概要

VELOCITY LOGGERに以下の2つの新機能を追加する構想：

1. **マルチロガー統合**: AIM、デジスパイス等の異なるデータロガーのデータを同一画面上で比較表示
2. **データマーケットプレイス**: セットアップデータ・ラップタイムデータのC2C売買機能

## 4. 法的リスク評価

### 4.1 データの所有権・知的財産 — リスク: 低

- 生のテレメトリデータ（ラップタイム、空気圧、温度等）は著作物に該当しない
- 自分の機材で取得したデータの売買は法的に問題なし
- 独自バイナリフォーマット（.drk, .xrk, .ld）の直接パースはリバースエンジニアリングに該当するリスクあり
- **対策**: CSV/JSON等のオープン形式のみを扱う

### 4.2 ロガーメーカーの利用規約

| メーカー | 規約 | 影響 |
|---|---|---|
| MoTeC | ソフト・コンテンツは個人非商用のみ | ユーザー記録データへの制約は明示なし |
| デジスパイス | 非商用個人利用。それ以外は事前連絡 | **事前確認を推奨** |
| AIM | 公開EULAなし | 製品付属規約の確認が必要 |

### 4.3 GPS・プライバシー（日本法）

- サーキットGPSデータ単体は個人情報に該当しない
- ドライバー名・車番との紐付けは個人情報保護法の対象
- **対策**: 販売データからドライバー特定情報を除外するオプション

### 4.4 商標

- 「AIM対応」等の表記は商標権リスク
- **対策**: 「CSVインポート（AIM Race Studio出力形式）」のような事実記載に留める

### 4.5 税務（日本）

| 項目 | 内容 |
|---|---|
| 所得区分 | 雑所得（副業）or 事業所得 |
| 確定申告 | 副業収入20万円/年超で必要 |
| 消費税 | 売上1,000万円以下は免税 |
| インボイス | 小規模なら不要 |

## 5. ビジネスモデル案

### 5.1 収益モデル: ハイブリッド（マーケットプレイス＋サブスク）

```
【無料ティア】
 ・自分のデータ記録・閲覧
 ・コミュニティデータの閲覧（一部）

【個別購入（C2C）】
 ・セットアップデータ: ¥500〜1,500/件
 ・ラップタイム+テレメトリ: ¥1,000〜3,000/件
 ・プラットフォーム手数料: 15〜20%

【月額サブスク】¥1,500〜3,000/月
 ・全データ閲覧し放題
 ・高度な比較・分析ツール
 ・AI分析アドバイス
```

### 5.2 価値の高いデータ

| データ種別 | 価値の理由 |
|---|---|
| 特定サーキット×特定車種のセットアップ | 同じ組み合わせのユーザーに直接的な価値 |
| プロ/上級ドライバーのデータ | コーチング的価値 |
| ウェット/特殊条件のセットアップ | 自力で試行錯誤するコストが高い |
| セットアップ変更前後の比較セット | 変更効果の実証データ |

### 5.3 段階的展開

```
Phase 0: コミュニティ育成（無料共有）
  → データ蓄積、ユーザー獲得、鶏と卵問題の解消

Phase 1: マーケットプレイス（C2C売買）
  → 手数料モデルでマネタイズ開始

Phase 2: サブスクリプション
  → 分析ツール強化、AI機能、プレミアム体験
```

## 6. リスクと課題

| リスク | 重大度 | 対策 |
|---|---|---|
| データ品質保証の困難さ | 高 | レビュー・評価システム、販売者実績表示 |
| 初期データ供給量不足（鶏と卵問題） | 高 | 無料共有コミュニティを先に育成 |
| サーキット×車種の組み合わせ過多 | 中 | 日本の人気サーキット×人気車種に集中 |
| デジスパイスの商用利用規約リスク | 中 | デジスパイス社への事前確認 |
| 不正データ・コピー品の流通 | 低〜中 | データ一意性検証、購入回数制限 |
| 各ロガーCSV仕様変更への追従 | 低 | パーサーの抽象化・モジュール化 |

## 7. 技術フィジビリティ調査結果（2026-06-03）

### 7.1 TrackDataAnalysis（OSS）ソースコード調査

| 項目 | 内容 |
|---|---|
| ライセンス | GPL-3.0（強コピーレフト）→ コード流用不可。アルゴリズム学習・クリーンルーム再実装は可 |
| 言語 | Python + Cython |
| 対応形式 | AIM .xrk, MoTeC .ld, VBOX, ECUMaster, RaceCapture, Race Technology, MegaLogViewer, iRacing（計8形式） |
| データモデル | Session → Laps → Channels（名前・単位・サンプルレート付き時系列配列）に正規化 |
| Web移植示唆 | ブラウザではDataView/ArrayBufferでバイナリ解析可能。大量データはLTTBダウンサンプリング+Web Worker推奨 |

### 7.2 AIM データ形式

AIM Race Studio 3のCSVエクスポート形式が判明:

```
"Format","AIM CSV File"
"Venue","鈴鹿サーキット"
"Date","2026/06/01"
"Time","10:30:00"
"Sample Rate",50
"Segment","Session1"

"Time","Distance","RPM","GPS_Speed","GPS_Latitude","GPS_Longitude",...
"s","km","rpm","km/h","deg","deg",...
0,0,3500,0.0,34.8431,136.5406,...
```

- メタデータ6行 → チャンネル名行 → 単位行 → データ行
- 時間: セッション開始からの経過秒
- チャンネル: ハードウェア構成により可変（固定スキーマなし）
- **パース難易度: 低**

### 7.3 デジスパイス データ形式

**重要な発見: 当初想定のCSV取り込みは不可**

| 項目 | 内容 |
|---|---|
| ネイティブ形式 | 非公開プロプライエタリバイナリ（仕様書なし） |
| CSV出力 | ラップサマリーのみ（ラップタイム・最高速度等）。生GPSデータは出力されない |
| 生データ出力 | **NMEAフォーマット（RMC文）のみ**。専用ツールで変換 |
| サンプルレート | DigiSpice IV: 最大20Hz / III: 最大10Hz |

NMEAパーサーのフィジビリティ:

| 項目 | 結論 |
|---|---|
| 実装難易度 | **低**（~50行のTypeScript、ライブラリ不要） |
| ブラウザ性能 | 問題なし（20Hz×30分=2.8MB、パース<50ms） |
| 導出可能データ | 速度、走行ライン、ラップタイム、縦G（高精度）、横G（中精度・フィルタ要） |
| 取得不可データ | 高度、衛星数（RMCのみのため） |
| デジスパイス固有の癖 | 時刻がJST（UTC+9）で出力される（NMEA規格違反）。要補正 |
